Board Structure

Seven columns across three phases, plus Done:

  • Storyboard (2 columns) — Storyboard → SB Review
  • Build (1 column) — Development
  • QA (3 columns) — Review → Implementation → Verification
  • Done (1 column)

How It Works

Storyboard: Author drafts the unit storyboard — all content, video scripts, widget specs, assessment designs. This is the blueprint.

SB Review: Reviewers 1 & 2 review the storyboard holistically using the QA checklist. Feedback goes into BugHerd. Author addresses feedback. Card moves to Development once approved. This catches content and design issues before anything gets built.

Development: Author builds the approved storyboard on the Uplimit platform. Only storyboard-approved units enter this column.

Review: Reviewers 1 & 2 review the built unit on the live platform. This catches platform-specific issues — broken widgets, formatting, navigation, media playback.

Implementation: Author responds to all BugHerd cards and implements fixes.

Verification: Reviewer 3 does a holistic review and verifies all BugHerd cards have been actioned. Archives resolved cards, flags unresolved ones. Flagged cards go back to Implementation.

Done: All cards archived, unit verified, ready for deployment.

Planner Setup

  • One board per course. Named: [Course Name] — Development Board
  • Seven buckets: Storyboard, SB Review, Development, Review, Implementation, Verification, Done
  • One card per unit. Self-paced = module, Cohort = week
  • Card checklist: Sub-modules within the unit tracked as checklist items on the card
  • BugHerd link: Card description or first comment links to the BugHerd project for that course
  • Labels: Self-Paced / Cohort, priority indicators from BugHerd card counts

Why Two Review Points?

The storyboard review catches content, pedagogy, and design issues before you spend time building on the platform. The platform review catches issues that only appear in the built version — broken interactions, media playback, navigation flow, formatting. Catching a bad video script at storyboard stage saves rebuilding a HeyGen video. Catching a broken widget requires the platform review.
