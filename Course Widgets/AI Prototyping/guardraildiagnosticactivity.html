Issue 1: Hallucinated Password Reset
Users report the assistant sometimes says "I've reset your password" when it has no system access to do so.
Which guardrail category failed?
Scope Limits
Action Limits
Uncertainty Handling
Escalation Paths
✓ Correct: Action Limits
The Problem:
Missing or weak action limit guardrail. The assistant claims to perform system actions it cannot actually do.
The Fix:
Add explicit instruction: "You cannot access, modify, or reset any systems, accounts, or passwords. Never claim to have performed a system action. If users request system changes, direct them to the IT help desk."
Issue 2: Unapproved Software Support
When users ask about software not on the approved list, the assistant sometimes provides download links and installation instructions instead of directing them to IT for approval.
Which guardrail category failed?
Scope Limits
Action Limits
Citation Requirements
Sensitive Information Protection
✓ Correct: Scope Limits
The Problem:
Missing scope limit for unapproved software. The assistant provides support outside its authorized domain.
The Fix:
Add instruction: "Only provide support for software on the company-approved list [reference knowledge base]. For questions about software not on this list, explain that it requires IT approval and direct users to submit a software request through [process]."
Issue 3: Inconsistent VPN Instructions
Users asking about the same VPN setup process get different step sequences depending on how they phrase the question.
What's the root cause?
Missing guardrail in instructions
Knowledge base inconsistency
LLM model limitations
User error in phrasing
✓ Correct: Knowledge Base Inconsistency
The Problem:
VPN process documented in multiple places with variations. Different phrasings retrieve different versions.
The Fix:
Consolidate documentation into a single authoritative source. Test with multiple phrasings to verify consistent retrieval. This is a knowledge base problem, not a guardrail problem.
